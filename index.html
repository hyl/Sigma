    Welcome to Sigma.

    Sigma is the easiest way to create a multi-user, one-on-one chat server using Node.js and WebSocket. Think Omegle, but less horny-teenagery and more open-sourcey. Sigma can be run with the included front end, or just a backend so that you are free to design your own UI. The included UI is based on Bootstrap.

    $ npm install crypto
$ npm install cli-color
$ npm install express
$ npm install websocket
$ node server.js

    You may want to edit server.js before running it so that you can change the port it runs on.

    Security

    When a user connects they are sent a message containing their ID, and an MD5 hash of a server-side salt, plus their ID:

    {"type": "id", "id": 1, "hash": "4b1ffd3c0c44a77c9121ffadcecedda5"}

    When they are assigned a partner, they are sent their partners ID and hash:

    {"type": "partner", "id": 2, "hash": "a3b170f94f79a0110ee96098cb0bb8dc", ...}

    Every request they make must contain their ID and hash, and if it interacts with their partner, their partners ID and hash. When requests are received the hash is regenerated and checked. If the two hashes do not match then the request is denied. This prevents a user just changing their ID and interacting with other users.

    Vulnerabilities

    While the system prevents a user chatting with someone they haven't spoken to, in theory if a user was to chat with someone they could make a note of the persons ID and hash, and then set the local partner variable to that and send messages with no problem (however the partner wouldn't be able to reply).
